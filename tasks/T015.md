## 验收标准（技术视角）

- [ ] **失败率统计实现**:
  - [ ] 实现 `GET /api/stats/failures`。
  - [ ] 支持按 `countryCode` 和 `aiPlatform` 聚合。
  - [ ] 返回字段: `totalJobs`, `failedJobs`, `successRate`, `topErrorReasons` (聚合 errorMessage)。
  - [ ] 时间窗口: 默认展示过去 24 小时的数据。
- [ ] **自适应重试策略优化**:
  - [ ] 修改 `BullMQ` 的 `defaultJobOptions`。
  - [ ] 对于 `RATE_LIMIT` (429) 错误，增加重试延迟 (指数退避指数系数调大)。
  - [ ] 对于 `AUTH_ERROR` 这种非暂时性错误，在 Worker 中使用 `UnrecoverableError` 主动终止重试，避免无效消耗。
- [ ] **超时与保护措施**:
  - [ ] 在 `src/lib/queue/index.ts` 中设置全局 Job `timeout` (例如 60s)。
  - [ ] 确保如果监测任务长时间无响应，系统能自动回收资源并标记为失败。
- [ ] **简单的报警机制 (日志级别)**:
  - [ ] 如果某一国家在短时间内失败率超过 50%，在服务器日志输出 `CRITICAL_GEO_FAILURE` 关键字。
