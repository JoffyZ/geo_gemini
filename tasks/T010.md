## 验收标准（技术视角）

- [x] **解析 Prompt 设计**:
  - [x] 包含详细的 JSON Output 指示（Zero-shot 或 Few-shot）。
  - [x] **品牌识别 (Mentions)**: 能够准确列出回答中提到的品牌名称。
  - [x] **排名 (Ranking)**: 如果是推荐列表，需识别品牌在回答中的相对排名。
  - [x] **情感分析 (Sentiment)**: 对品牌提到的语境进行 Positive/Negative/Neutral 判定 。
  - [x] **引用来源 (Sources)**: 如果 AI 提供了链接（如 Perplexity），解析并返回 sources 数组。
- [x] **Parser 服务逻辑**:
  - [x] `LLMParser.parse(rawText: string, model: string)` 方法。(Implemented as `parseAIResponse`)
  - [x] 使用 Zod 对 LLM 返回的 JSON 字符串进行验证（Runtime Validation）。
  - [x] **容错处理**: 即使 LLM 输出了非标准 JSON，也应通过正则尝试提取 JSON 块。(OpenAI Structured Output minimizes this need).
- [x] **国家维度感知**:
  - [x] 解析逻辑在必要时应考虑 `countryCode` 上下文（例如不同地区的品牌译名差异）。
- [x] **测试验证**:
  - [x] 针对已有的 OpenAI/Perplexity 测试响应，验证解析出来的 JSON 字段与 `src/db/schema.ts` 定义的 `content` 结构完全对齐。
