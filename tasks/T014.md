## 输入

- BullMQ Job Events (`completed`, `failed`)。
- `monitoringWorker` 运行时上下文。
- `src/db/schema.ts`: Drizzle 模型定义。

---

## 输出

- `monitoring_job_logs` 表 (DB): 详细的任务运行日志。
- `src/lib/logger.ts`: 统一的监控记录服务。

---

## 验收标准（技术视角）

- [x] **数据库模型定义**:
  - [x] 字段: `id`, `tenantId`, `promptId`, `aiPlatform`, `countryCode`, `status` (enum: pending, success, failed)。
  - [x] 字段: `durationMs` (执行耗时), `errorMessage`, `errorStack` (存储堆栈信息)。
  - [x] 字段: `createdAt`。
- [ ] **Worker 日志打点集成**:
  - [ ] 成功时: 在 Worker 逻辑末尾将成功状态持久化到 `monitoring_job_logs`。
  - [ ] 失败时: 在 Worker 的 `catch` 块中捕获详细 Error 并持久化到 `monitoring_job_logs`。
  - [ ] 记录任务的 `processAt` 和 `finishedAt` 的时间戳，并计算 `durationMs`。
- [ ] **错误分类标识**:
  - [ ] 识别并区分以下错误类型: `AUTH_ERROR`, `RATE_LIMIT`, `LLM_TIMEOUT`, `PARSING_ERROR`, `NETWORK_GEO_ERROR`。
  - [ ] 将错误代码存储在 `errorMessage` 前缀中，便于后续聚合统计。
- [ ] **事务一致性**:
  - [ ] 记录日志的操作不应导致主监测流程中断 (使用 `async/await` 但确保异常隔离)。
