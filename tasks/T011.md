## 验收标准（技术视角）

- [ ] **UI 表单设计**:
  - [ ] **AI 平台选择**: 下拉框，包含 chatgpt, perplexity, gemini, google_ai_overviews。
  - [ ] **国家选择**: 下拉框，支持常用国家（US, CN, UK 等）。
  - [ ] **Prompt 输入**: 大文本框，支持多行输入。
  - [ ] **即时测试按钮**: 点击后调用 API。
- [ ] **后端接口逻辑**:
  - [ ] `POST /api/explorer`: 接收平台、国家、Prompt。
  - [ ] 流程: 工厂实例化适配器 -> 调用 `query` 获取原始回答 -> 调用 `LLMParser.parse` 获取结构化结果。
  - [ ] 返回原始回答 + 解析后的 JSON + 执行耗时。
- [ ] **结果展示**:
  - [ ] **Raw Output**: 源代码/文本块展示。
  - [ ] **Structured Preview**: 结构化数据的卡片化展示（例如：提及的品牌列表、识别的情感）。
  - [ ] **错误状态**: 清晰展示 API 调用失败的错误信息。
- [ ] **交互逻辑**:
  - [ ] 状态管理: 处理 Loading 状态和多个请求的顺序。
