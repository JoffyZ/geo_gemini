## 输入

- 环境变量: `CRON_SECRET` (用于验证 Vercel Cron 触发安全)。
- 预定义国家列表: `['US', 'GB', 'CN']` (暂定)。
- 预定义 AI 平台列表: `['chatgpt', 'perplexity', 'gemini', 'google_ai_overviews']`。

## 验收标准（技术视角）

- [ ] **接口安全控制**:
  - [ ] 校验 `Authorization: Bearer ${CRON_SECRET}` 头部。
  - [ ] 仅允许 `GET` 请求 (Vercel Cron 默认行为)。
- [ ] **任务分发逻辑**:
  - [ ] 获取所有 `isActive: true` 的 Prompts。
  - [ ] 遍历 Prompts × Target Countries × Target Platforms 的笛卡尔积。
  - [ ] 将每个三元组作为 Job Data 加入 `monitoringQueue`。
  - [ ] Job Data 包含: `promptId`, `countryCode`, `aiPlatform`, `tenantId`。
- [ ] **分发性能优化**:
  - [ ] 使用 `bulk` 添加 (如果 BullMQ 版本支持 `addBulk`) 或 Promise.all 并行添加，避免循环阻塞。
  - [ ] 防止重复分发 (使用 Job ID: `${promptId}-${countryCode}-${platform}-${date}`)。
- [ ] **日志记录**:
  - [ ] 记录本次 Dispatcher 运行生成的总任务数。
  - [ ] 记录分发耗时。
