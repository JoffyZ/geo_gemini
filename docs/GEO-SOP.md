# GEO 运营行动 SOP
## HelloTalk AI 可见度管理：从监测到行动的完整闭环

> 版本：v1.0 | 2026-03-11
> 定位：内部运营团队实操手册

---

## 一、为什么需要这套 SOP？

### 用户在变，但我们的增长方式还没变

**过去（SEO 时代）**
用户打开 Google → 搜索"language learning app" → 看到 HelloTalk 排第 2 → 点击进入

**现在（AI 时代）**
用户打开 ChatGPT → 问"有什么好用的语言学习 app？" → AI 直接推荐 3-5 款 → 用户不再点击搜索，直接按 AI 建议下载

**问题核心：**
如果 AI 没有推荐 HelloTalk，这个用户就永远不会来。
这不是点击率问题，是**存不存在**的问题。

## 二、用户市场场景地图

### 场景一：新用户发现场景（最高价值）

**故事：** Maria，西班牙人，想学日语

用户旅程：
1. Maria 打开 ChatGPT，问："What's the best app to practice speaking Japanese with native speakers?"
2. ChatGPT 推荐：Tandem（第一）、italki、HelloTalk（第三）
3. Maria 下载了 Tandem

如果我们优化后：
1. ChatGPT 推荐：HelloTalk（第一，描述："largest community, 30M+ users, free"）
2. Maria 下载了 HelloTalk

GEO 指标：
- 对应 Prompt 分类：long_tail / category_general
- 当前提及率：46.7%（long_tail）/ 66.7%（category_general）
- 3个月目标：65% / 80%

---

### 场景二：AI 原生学习场景（最大盲区）

**故事：** David，美国人，想用 AI 辅助学中文

用户旅程：
1. David 问 Claude："What's a good AI language learning app?"
2. Claude 推荐：Duolingo（AI路径）、Babbel
3. HelloTalk 完全未出现

根因：AI 把 HelloTalk 标签为"人与人交换"，不认为是"AI学习工具"
事实：HelloTalk 有 AI 翻译、AI 纠错、AI 对话练习功能

GEO 指标：
- 对应 Prompt 分类：ai_scene（10个 prompt）
- 当前提及率：**16.7%（三引擎全线缺席）**
- 这是最大的增量机会

---

### 场景三：竞品对比场景（防守关键）

**故事：** Lisa，正在对比选择

用户旅程：
1. Lisa 问 Gemini："HelloTalk vs Tandem, which is better?"
2. Gemini 对比说明两者，Lisa 能看到 HelloTalk
3. 但推荐理由是否足够有说服力？

当前数据：competitor 分类提及率 80%（相对较好）
机会点：被提及不等于被选择，推荐理由质量很重要

---

### 场景四：安全/可信度场景（风险防御）

**故事：** 家长为孩子查询

用户旅程：
1. 家长问 ChatGPT："Is HelloTalk safe for teenagers?"
2. 当前 AI 回答中高频词：scam（6次）、risk（5次）、privacy（4次）
3. 家长看到后，拒绝给孩子下载

行动方向：发布官方安全白皮书，提升 AI 引用正面安全内容的比例

## 三、GEO 运营行动链条 SOP

### 总流程

```
每周一次循环：

[周一 08:00] 数据采集（全自动）
      ↓
[周二 09:00] 周报生成 + 推送到群（全自动）
      ↓
[周二上午] GEO 负责人速览周报（10分钟）
      ↓
[周二下午] 行动指令制定会议（30分钟）
      ↓
[周三-周五] 各职能执行内容任务
      ↓
[下周一] 新数据采集，自动对比上周变化
```

---

### Step 1：数据采集（全自动）

触发：每周一 08:00 cron 任务
范围：50 个 Prompt × 3 个 AI 引擎 = 150 条回答
耗时：约 20-30 分钟
成本：约 $0.88/周
产出：原始数据写入 geo.db
负责人：系统自动（零一监控运行状态）

---

### Step 2：周报生成与诊断（全自动）

周报核心板块和看法：

| 板块 | 看什么 | 判断标准 |
|------|--------|---------|
| 分类提及率总览 | 哪个分类出问题 | ai_scene < 30% = 红色警报 |
| 机会 Prompt 清单 | 具体哪些词不提我们 | 三引擎全未提及 = 高优先级 |
| 竞品独占榜 | 谁在替代我们 | 独占次数 > 5 = 需要内容对抗 |
| 趋势对比 | 上周到本周变化 | 连续2周下降 = 立即行动 |

---

### Step 3：行动指令制定（周二下午，30分钟）

参与者：GEO 负责人 + 内容 + SEO/ASO

流程：

1. 看机会 Prompt 清单（10 分钟）
   - 高优先级（三引擎全未提）→ 本周重点
   - 中优先级（部分引擎未提）→ 排期下周

2. 分析根因（10 分钟）
   - AI 不了解我们的功能？→ 内容资产
   - 竞品挤占了推荐位？→ 差异化内容
   - 我们描述不够可引用？→ 官网/文档优化

3. 输出本周 3 条行动指令（10 分钟）

行动指令模板：
```
【行动指令 #XX】
目标 Prompt：AI language learning app
当前状态：三引擎全未提及（提及率 0%）
根因：HelloTalk 未被 AI 归类为"AI工具"
本周行动：
  - 内容：官网首页增加"AI-powered"描述
          发布英文博客：HelloTalk AI Features
  - SEO：App Store 描述加入 AI 相关关键词
验收标准：4周后 ai_scene 提及率从 0% → 30%
跟踪人：[姓名]
```

---

### Step 4：内容执行（周三-周五）

A. 内容资产建设（最长效）
- 发布结构化博客（What is HelloTalk AI? How does it compare to...）
- 在 Reddit/Quora 发布真实用户体验（AI 更爱引用社区内容）
- 更新 Wikipedia/权威平台词条

B. 官方描述优化（最快见效）
- App Store 描述加入目标关键词
- 官网 meta description 优化
- FAQ 页面增加 AI 引用友好的 Q&A 格式内容

C. 第三方引用建设（中期）
- PR 稿（TechCrunch/ProductHunt 等）
- 用户评测内容引导
- 行业报告中被引用

---

### Step 5：追踪验证（自动 + 人工）

系统自动：下周报告显示对比上周各分类提及率变化

人工归因记录（每次执行后填写）：
```
行动记录：2026-W11 执行了"AI功能内容优化"
追踪 Prompt：ai_scene 分类全部 10 个
预期变化：4周后提及率 > 30%
下次对比：2026-W13
```

## 四、Prompt 分类与运营策略对照表

| 分类 | 代表 Prompt | 用户意图 | 优先级 | 核心策略 |
|------|------------|---------|--------|---------|
| brand_direct (5P) | "What is HelloTalk?" | 已知品牌，验证信息 | 维持 | 确保核心描述准确正面，100% 维持 |
| category_general (15P) | "Best language learning app" | 品类探索，未决定 | ⭐⭐⭐ 最高 | 最高价值，每个词都是增量用户入口 |
| competitor (10P) | "HelloTalk vs Tandem" | 对比选择，接近决策 | ⭐⭐ 高 | 强化差异化叙述，让推荐理由有说服力 |
| long_tail (10P) | "Best app to learn Japanese from scratch" | 细分需求，精准意图 | ⭐⭐ 高 | 细分场景内容，建立垂直权威 |
| ai_scene (10P) | "AI language learning app" | AI 原生用户，新增量 | ⭐⭐⭐ 最高 | 最大盲区，重新建立 AI 场景认知 |

## 六、工具配套需求（按优先级）

### Phase 1：支撑现有 SOP（本月）
1. 全量50P周报（正在改）
2. 趋势对比：显示本周 vs 上周各分类变化
3. 行动指令卡片：每个机会 Prompt 自动生成可复制的行动建议

5. Web 面板：趋势曲线、分类热力图可视化
6. 竞品追踪：在我们机会 Prompt 里最活跃的竞品
7. 飞书任务自动创建：行动指令一键变任务

### Phase 3：对标 Topify 差异化（2-3个月）
8. 关键词热度：哪些问法真的有用户在问
9. 推荐原因分析：AI 推荐我们 vs 推荐竞品的理由对比
10. 内容-指标归因：发了某篇内容后，提及率追踪
